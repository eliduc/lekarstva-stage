# Лекарства — помощник

Веб-приложение (PWA) для приёма лекарств пожилым человеком и контроля со стороны
ухаживающего. Крупный шрифт, пошаговый мастер выдачи, таблетница, будильники,
4 языка (ru/en/he/uz, с RTL для иврита), Telegram-уведомления и распознавание
упаковок/рецептов через Claude.

**Приложение:** https://eliduc.github.io/lekarstva/  *(включается после публикации)*

## Архитектура (клиент-сервер, как в [metapel](https://github.com/eliduc/metapel))

Статический клиент на GitHub Pages + приватный репозиторий GitHub в роли «сервера»
(хранилище данных через Contents API). Своего бэкенда нет.

```
index.html        оболочка (header, навигация, экраны, модалки, будильники)
css/styles.css    стили
js/i18n.js         переводы и форматирование (без DOM)
js/data.js         типы лекарств, дефолтные данные, иконки, хелперы (без DOM)
js/storage.js      MedStore — слой localStorage
js/sync.js         MedSync — синхронизация через GitHub Contents API
js/app.js          контроллер: экраны, мастер, напоминания, Telegram, ИИ, sync
img/, icons/       фото лекарств и иконки приложения
sw.js, manifest.json   PWA: офлайн-кэш и установка «на экран Домой»
```

Загрузка скриптов: `i18n → data → storage → sync → app`.

## Синхронизация (конфиг + живой статус)

Два независимых канала в приватном репозитории данных:

- **`backup/data.json`** — конфигурация (лекарства, расписание, времена, настройки,
  хэш пароля). Версия (`generation`) + compare-and-swap (`sha`) защищают от затирания
  более свежей копии. Подтягивается при запуске и по таймеру, заливается после изменений.
- **`status/live.json`** — «живой» статус: сегодняшние отметки приёма (`slots`,
  привязаны к дате) и недельная таблетница (`box`). Слияние «последняя запись
  по времени» (LWW) — планшет пациента и телефон ухаживающего пишут одновременно
  без потерь. Опрашивается каждые ~20 с → ухаживающий почти в реальном времени
  видит, принял ли пациент таблетки.

Секреты (токен GitHub, ключ Anthropic, имя устройства) в облако не уходят.

## Настройка «сервера»

1. Откройте приложение → **Настройки** (пароль по умолчанию `1234`, меняется там же).
2. Раздел **☁ Синхронизация (GitHub)**:
   - **Репозиторий данных** — `owner/имя` приватного репо (например `eliduc/lekarstva-data`).
   - **Токен доступа** — fine-grained PAT: github.com → Settings → Developer settings →
     Fine-grained tokens; доступ только к этому репозиторию, право **Contents: Read and write**.
   - **Имя устройства** — «Планшет» / «Телефон» (видно ухаживающему в живом статусе).
   - Включите тумблер. Токен хранится только на устройстве.
3. На втором устройстве введите тот же репозиторий и токен — данные синхронизируются.

## Распознавание (ИИ)

Раздел **🤖 Распознавание** в настройках: введите ключ Anthropic API
(`sk-ant-...`). Запросы идут напрямую в `api.anthropic.com`
(`anthropic-dangerous-direct-browser-access`), модель `claude-sonnet-4-6`.
Ключ хранится только на устройстве. Без ключа кнопки сканирования сообщают,
что нужен ключ.

## Разработка

Статические файлы, без сборки. Локальный запуск:

```
python -m http.server 8000
# открыть http://localhost:8000
```

Ассеты (CSS, фото лекарств, иконки, модули i18n/data) извлечены из исходного
одностраничного `Помощник_лекарства.html` скриптами `tools/extract_assets.py`
и `tools/extract_app.py` (разовый шаг, повторно не требуется).

## Ограничения

Напоминания работают, пока вкладка/приложение открыты (`setInterval`, как в исходнике) —
это не системные фоновые push. Частичную подстраховку даёт Telegram и живой статус.
